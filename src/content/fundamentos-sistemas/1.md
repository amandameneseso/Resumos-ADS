# Linux

## 1. Linux

- **Definição e História:**
    - **Linux é um sistema operacional de código aberto**.
    - Foi criado por **Linus Torvalds em 1991**, quando ele era estudante na Universidade de Helsinque e precisava de um sistema operacional para seu computador pessoal.
    - Linus se **inspirou no Unix** e começou o projeto como um hobby, que rapidamente se expandiu com a contribuição de outros desenvolvedores.
    - O **kernel do Linux foi lançado como software livre**, permitindo que qualquer pessoa veja, modifique e distribua o código-fonte. Essa abertura contribuiu para que o Linux se tornasse uma das principais opções de sistema operacional para servidores e computadores pessoais.
    - Uma frase famosa de Linus Torvalds, "Se você não pode explicar algo de forma simples, você não entende o assunto o suficiente", reflete a filosofia de simplicidade do Linux.
    - **Diferença entre Unix e Linux**: O Unix é proprietário, enquanto o Linux é de código aberto, sendo considerado mais flexível e personalizável.
- **Mascote:** O mascote do Linux é **Tux, um pinguim**, criado por Larry Ewing em 1996 e frequentemente usado como logotipo de diversas distribuições.
- **Licença:** O Linux é de código aberto graças à licença **GPL (General Public License)**, criada por Richard Stallman, fundador do Projeto GNU.
- **Kernel:**
    - **Kernel** é o **núcleo** do sistema operacional, responsável por **gerenciar tarefas básicas** do computador, como gerenciamento de memória e arquivos.
    - Porém, o kernel por si só não é suficiente para um sistema operacional completo; ele precisa de **softwares adicionais**, como gerenciadores de janelas, editores de texto e navegadores, que, juntos, formam as **distribuições Linux** (como o Ubuntu, Debian e Fedora).
    - Cada distribuição Linux tem suas próprias características e conjunto de softwares adicionais, permitindo que os usuários escolham a que melhor se adapta às suas necessidades.
    - As **principais funcionalidades do kernel** incluem: gerenciamento de memória, gerenciamento de processos (tarefas dos programas), gerenciamento de arquivos (criação, leitura, escrita, exclusão) e gerenciamento de dispositivos (discos rígidos, impressoras, USB).
    
    ![image.png](https://paperx-dex-assets.s3.sa-east-1.amazonaws.com/images/1623268520925-Ulrh00jiQU.png)
    
- **Projeto GNU:**
    - É um projeto de **software livre criado em 1983 por Richard Stallman**, programador, escritor e ativista político americano.
    - Stallman também é o criador da Fundação para o Software Livre.
    - O objetivo do projeto GNU é desenvolver um **sistema operacional completo, livre de restrições de uso e modificação, e de código fonte aberto**, semelhante ao Unix.
    - O projeto GNU foi **fundamental para o desenvolvimento do Linux**, pois muitos de seus componentes foram desenvolvidos como parte do GNU.
    - Existem **distribuições Linux construídas inteiramente com componentes GNU**, conhecidas como distribuições GNU/Linux, como o Debian. No entanto, nem todo Linux é composto apenas por componentes GNU.
- **Filosofia Linux:**
    - A filosofia do Linux é **muito semelhante à do software livre**, baseada no princípio de que o software deve ser **livre para ser usado, estudado, modificado e distribuído**.
    - Isso contrasta com o software proprietário, onde os usuários não podem ver ou modificar o código-fonte.
    - Outra parte importante da filosofia é a **colaboração e a comunidade**, onde muitas pessoas trabalham juntas para melhorar o software.
- **Aplicações de Linux:** O Linux é um sistema operacional **versátil e utilizado em diversas aplicações.**
    - **Servidores:** Devido à sua estabilidade, segurança e flexibilidade, é amplamente usado para web hosting, e-mail, arquivos e banco de dados, inclusive em sistemas de nuvem como Amazon Web Services e Google Cloud.
    - **Dispositivos Móveis:** Muitos fabricantes usam Linux, incluindo o **Android**, que é baseado em Linux. Também é encontrado em Smart TVs, dispositivos de streaming (Chromecast) e Raspberry Pi.
    - **Aplicações Industriais:** Usado em sistemas de automação industrial, sistemas de transporte inteligentes e Internet das Coisas (IoT), devido à sua estabilidade, segurança e adaptabilidade.
- **Fundações ligadas ao Linux:**
    - **Linux Foundation:** Organização sem fins lucrativos fundada em 2000, que promove e protege o Linux, patrocinando o desenvolvimento do kernel, organizando eventos e fornecendo recursos para empresas.
    - **Free Software Foundation (FSF):** Fundada por Richard Stallman em 1985, com o objetivo de promover o software livre e garantir a liberdade dos usuários. A FSF criou a licença GPL e patrocina o desenvolvimento de projetos de software livre, incluindo o Projeto GNU.

## 2. Licenças de software

**Licenças de software** são contratos legais que **regulam a utilização, distribuição e modificação de software**, definindo condições e restrições para usuários e responsabilidades de desenvolvedores.

- **Software Livre:**
    - Software cujo **código-fonte está disponível para ser modificado e distribuído livremente** pelos usuários.
    - Usuários podem fazer alterações, distribuir e até vender suas versões, desde que respeitem as condições da licença.
    - **Exemplos:** Linux, Firefox, LibreOffice, MySQL.
    - A filosofia é que todos devem ter acesso ao código-fonte e poder modificá-lo e distribuí-lo sem restrições.
- **Open Source:**
    - Softwares cujos **códigos-fontes estão disponíveis para serem estudados, modificados e distribuídos**.
    - A **principal diferença para o software livre** é que o software livre foca na liberdade de uso, estudo, modificação e distribuição, enquanto o open source foca na disponibilidade do código-fonte para colaboração e melhorias.
    - Nem todo software open source é livre, pois alguns podem ter restrições de modificação/distribuição ou ser vendidos comercialmente.
    - A filosofia do open source é que o compartilhamento do código-fonte permite a **colaboração** e o desenvolvimento de software mais inovador e de qualidade.
    - **Exemplos:** Android, Blender, Redmine.
- **Copyleft:**
    - Categoria especial de **licenças de software livre que garantem que as modificações e distribuições derivadas do software mantenham as mesmas condições** (para que as mesmas liberdades sejam preservadas nas versões derivadas).
    - Exige que quem distribua o software ou suas modificações forneça o código-fonte e licencie sob os mesmos termos da licença original.
    - **Exemplos de licenças copyleft:** GPL v2 (usada no Linux), LGPL, AGPL.
- **Freeware e Shareware:**
    - **Freeware:** Software **gratuito, sem custo**, que pode ser usado, copiado e distribuído livremente, mas o código-fonte geralmente **não está disponível** para modificação. **Exemplos:** Firefox, Notepad++, Avast.
    - **Shareware:** Software **distribuído gratuitamente para teste e avaliação**, mas que geralmente **requer uma licença paga para uso permanente e/ou integralmente. Exemplo:** Adobe Photoshop.
- **Software Proprietário:**
    - Software **controlado e licenciado por uma empresa ou indivíduo**, que tem o direito exclusivo de distribuir, modificar e usar.
    - Geralmente vem com **licença restritiva que proíbe modificação, distribuição e uso compartilhado**, e o código-fonte **não está disponível**.
    - A aquisição da licença dá direito de uso, mas não de modificação do código-fonte.
    - **Exemplos:** Windows da Microsoft, Adobe Premiere, Microsoft Project.
- **Software as a Service (SAAS):**
    - Modelo de entrega de software onde as **aplicações são acessadas pela internet**, geralmente por assinatura.
    - Os usuários acessam o software e seus dados via navegador web ou aplicativos, com os dados armazenados em servidores remotos.
    - **Exemplos:** Google Apps, Salesforce, Dropbox, Netflix, Spotify, Slack. Este modelo de negócio tem crescido globalmente.

## 3. Distribuições Linux

- Uma **distribuição Linux é uma versão do sistema operacional Linux que inclui um conjunto de ferramentas e programas adicionais**.
- São criadas por diferentes organizações e comunidades, com objetivos e características próprias.
- Algumas são para desktops/laptops, outras para servidores ou dispositivos embarcados.
- Todas as distribuições usam o **mesmo kernel Linux**, mas cada uma tem sua filosofia e conjunto de ferramentas.
- Podem ser gratuitas e de código aberto, ou comerciais com suporte pago.
- **Principais famílias e distribuições:**

![image.png](https://files.passeidireto.com/787100f8-e7f4-40e4-8fc7-d0c72a1de239/bgb.png)

- **Debian:**
    - Uma das **mais antigas e respeitadas**, criada em 1993 por Ian Murdock para ser um sistema operacional livre e de código aberto.
    - Principal característica: **gerenciador de pacotes APT (Advanced Package Tool - .deb)**, que facilita a instalação, atualização e remoção de software.
    - Projeto **Debian/Hurd:** visa uma versão baseada no kernel Hurd (Sistema de núcleos da GNU), projetado para ser mais seguro e escalável que o Linux.
    - Vem com diferentes interfaces gráficas, sendo o **GNOME a principal**.
    - Deu origem a outras distribuições como **Ubuntu, Mint e Kurumin (brasileira)**.
    - **Raspbian** é uma variação otimizada para Raspberry Pi.
- **Ubuntu:**
    - Sistema operacional baseado em Linux desenvolvido pela empresa sul-africana Canonical.
    - Projetado para ser **fácil de usar e acessível para iniciantes e avançados**.
    - Baseado no Debian e usa o gerenciador de pacotes *apt*.
    - O **comando sudo surgiu com o Ubuntu**, usado para dar privilégios de administrador.
    - Compatível com **WSL (Subsistema Windows para Linux)** no Windows 10, permitindo executar comandos Linux diretamente no Windows sem precisar de uma máquina virtual ou dual boot. Isso pode ser útil para desenvolvedores que precisam alternar entre o Windows e o Linux.
- **Mint:**
    - Distribuição Linux **baseada no Ubuntu**.
    - Criada para fornecer uma **experiência de usuário similar ao Windows**, ideal para quem está migrando.
    - Inclui o ambiente de desktop **Cinnamon**, com barra de tarefas e menu iniciar semelhantes ao Windows. Vem com aplicativos populares pré-instalados, como LibreOffice e Firefox.
- **Red Hat:**
    - Amplamente **utilizada em empresas e organizações**.
    - Principal característica: **gerenciador de pacotes RPM (Red Hat Package Manager)**, que garante conformidade com políticas de segurança e estabilidade.
    - **Não é gratuita**; é disponibilizada sob licença comercial, exigindo pagamento por suporte e atualizações de segurança. No entanto, pode ser baixada e usada gratuitamente sem suporte.

## 4. Terminologia Linux

- **Processo de Boot:**
    - Sequência de etapas que ocorrem ao ligar o computador.
    - Começa com o carregamento do **firmware (BIOS ou UEFI)**, que controla funções básicas do hardware.
    - O firmware verifica se o disco rígido e outros dispositivos de armazenamento estão conectados e funcionando corretamente, e então procura por um sistema operacional para carregar (boot loader). O arquivo de boot loader no Linux geralmente é o **GRUB**.
    - O GRUB carrega o kernel Linux, que então inicia o sistema operacional, módulos de dispositivos, configurações de hardware, serviços do sistema e, finalmente, a interface gráfica ou o prompt de comando.
    - **Boot loaders comuns no Linux:** GRUB (padrão), LILO e ISOLINUX. O GRUB oferece interface amigável e recursos avançados, como edição da linha de comando do kernel.
    
    ![image.png](https://files.passeidireto.com/f0c24307-1584-40a3-8441-4d145f5a6bf4/bge.png)
    
- **Sistema de arquivos:**
    - Estruturas de dados que organizam e gerenciam arquivos em dispositivos de armazenamento.
    - Responsáveis por criar, ler, atualizar e excluir arquivos, além de gerenciar permissões.
    - **Sistemas de arquivos comuns no Linux:**
        - **Ext4:** Amplamente utilizado, padrão na maioria das distribuições modernas, bom desempenho e capacidade (até 16 TB).
        - **XFS:** Popular para sistemas com arquivos gigantescos, suporta até 9 EB, usado em servidores de arquivos e sistemas de armazenamento de massa (big data).
        - **Btrfs:** Sistema de nova geração com recursos avançados como snapshots, replicação e balanceamento de carga, boa opção para alta capacidade e recursos avançados.
- **X Window System e a Interface Gráfica Linux:**
    - **X Window System (X ou X11):** Software de sistema para gerenciamento de janelas para sistemas Unix-like (como Linux), fornecendo uma camada para aplicativos gráficos criarem e gerenciarem janelas, menus e outros elementos gráficos.
    - O X Window System é a **camada inferior da Graphic User Interface (GUI)** do Linux.
    - Ambientes de desktop e gerenciadores de janelas mais comuns: GNOME e KDE.
- **Terminal (Linha de comando):**
    - Interface que permite o usuário **interagir com o sistema operacional por comandos digitados em uma interface de texto**.
    - É uma das formas mais **poderosas e flexíveis** de trabalhar com Linux, permitindo tarefas avançadas como gerenciar arquivos, configurar o sistema, instalar/desinstalar programas.
    - Comandos podem ser **automatizados via scripts** para tarefas repetitivas ou complexas.
- **Shell:**
    - O shell é a **interface de acesso ao sistema operacional**, onde é possível interagir com o sistema por meio de **comandos**.
        
        ![image.png](https://www.certificacaolinux.com.br/wp-content/uploads/2020/04/image-1.png.webp)
        
    - No Linux, existem vários tipos de shells, cada um com suas características e comandos. **Exemplos de shells comuns:** Bash (padrão na maioria das distribuições), Zsh (Z Shell) e Tcsh (Tenex C Shell).

## 5. Serviços no Linux e certificações

- **Serviços no Linux:**
    - Programas instalados no Linux para realizar determinadas tarefas. **Exemplos de serviços populares:** LibreOffice, Mozilla Firefox, ****GIMP, player de vídeos VLC e gravador de vídeos OBS Studio.
- **Jogos no Linux:**
    - Existem muitos jogos disponíveis, desde indies até AAA.
    - Alguns jogos não são nativos e precisam de **software de emulação**, como:
        - **Wine:** Permite executar programas do Windows no Linux, compatível com ampla variedade de jogos e aplicativos.
        - **Proton:** Versão do Wine desenvolvida pela Valve (criadora da Steam) para melhorar a compatibilidade de jogos do Windows com Linux, com configurações otimizadas e mais fácil de usar.
- **Certificações Linux:**
    - Excelente forma de **demonstrar habilidades e conhecimentos** em um campo ou tecnologia, frequentemente exigidas por empregadores e abrindo portas para oportunidades de carreira.
    - **Linux Professional Institute (LPI):** Organização sem fins lucrativos mundialmente reconhecida que oferece certificações para profissionais de sistemas Linux.
    - A LPI oferece quatro níveis de certificação: Linux Essentials, LPIC-1, LPIC-2 e LPIC-3. A Linux Essentials não é obrigatória, e foi feita para quem não conhece nada de Linux. O LPIC-1 é a certificação básica e abrange os conhecimentos necessários para instalar, configurar, usar o terminal e administrar sistemas Linux. O LPIC-2 é a certificação intermediária, e a LPIC-3 é a certificação avançada e abrange tópicos especializados, como virtualização e segurança avançada.
